# ⚓ Simple Sea Battle (WinForms V_0)

> **Учебный проект.** Быстрый перенос ядра «Морского боя» на Windows Forms — без архитектурных паттернов.

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

---

## 📖 О проекте

Данный проект — **вторая итерация** разработки игры «Морской бой».  
После создания [ядра игры в консольном приложении](https://github.com/VladimirRepp/CS---Quest-Learn/tree/main/Samples/SimpleSeaBatlle/GameCoreSimpleSeaBattle), логика **быстро перенесена на визуальный интерфейс Windows Forms**.

Проект создан в обучающих целях и демонстрирует **промежуточный этап**: как можно оперативно подключить готовое ядро к графическому интерфейсу.

---

## 🎯 Цели и задачи

- ✅ Перенести существующее ядро игры «Морской бой» на Windows Forms
- ✅ Показать, как быстро можно подключить логику к визуальному интерфейсу
- ✅ Реализовать кликабельное игровое поле и визуальную обратную связь
- ⚠️ **Важно:** данный пример **не использует архитектурных паттернов** — это намеренно упрощённая версия

---

## 🧱 Что под капотом?

Проект использует **то же ядро**, что и консольная версия:

| Сущность | Назначение |
|----------|------------|
| `GameMap` | Игровое поле 5×5, случайная генерация корабля, проверка клеток |
| `GameLogic` | Обработка выстрелов, проверка попадания, определение победы |

Логика осталась **нетронутой** — изменился только способ взаимодействия с пользователем.

---

## ⚠️ Почему «без архитектуры»?

Эта версия создана осознанно простой.  
Логика и интерфейс здесь намеренно **не разделены по паттернам** — это **отправная точка**, чтобы в следующей итерации наглядно показать:

- 🧪 Сложности тестирования такого кода
- 🔧 Проблемы при модификации и расширении
- 📈 Преимущества внедрения паттерна **MVP**

> *«Чтобы оценить преимущества архитектуры, нужно сначала увидеть код без неё.»*
