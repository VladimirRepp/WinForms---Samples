# ⚓ Simple Sea Battle (Core Game / Console)

> **Учебный проект**, демонстрирующий важность разделения бизнес-логики и пользовательского интерфейса.

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

---

## 📖 О проекте

Проект создан в обучающих целях в рамках изучения архитектуры программного обеспечения.

Ключевая задача — показать, что **ядро игры не должно зависеть от интерфейса**.  
Данный проект реализует игровую логику «Морского боя» в **чистом виде**, а консоль используется исключительно для **быстрых тестов корректности работы логики**.

Игровая логика спроектирована таким образом, чтобы в дальнейшем её можно было легко подключить к любому визуальному интерфейсу: **WinForms**, **WPF** или **Unity**.

---

## 🎯 Цели и задачи

- ✅ Показать, почему нельзя смешивать бизнес-логику и слой представления
- ✅ Реализовать переиспользуемое ядро игры «Морской бой»
- ✅ Использовать консоль как инструмент для быстрого тестирования логики
- ✅ Подготовить основу для переноса на графические интерфейсы: **WinForms**, **WPF**, **Unity**

## 📜 Правила игры

| Параметр | Описание |
|----------|----------|
| Размер поля | 5×5 клеток |
| Корабли | Один однопалубный корабль |
| Генерация | Случайная расстановка корабля |
| Игровой процесс | Игрок делает выстрелы по координатам |
| Победа | Наступает при первом попадании в корабль |

### Реализовано

- 🎲 Случайная генерация позиции корабля
- 🎯 Обработка выстрелов игрока
- 🏆 Определение победы при попадании

## 🧱 Архитектура

Проект построен на **независимом игровом ядре**, разделённом на логические сущности:

| Сущность | Назначение |
|----------|------------|
| `GameMap` | Игровое поле: хранение состояния клеток, размещение корабля, проверка занятости |
| `GameLogic` | Основная игровая логика: обработка выстрелов, проверка попадания, определение победы |

**Главный принцип:**  
Логика **полностью независима** от интерфейса. Ядро ничего не знает о том, как оно отображается — в консоли, окне WinForms, на сцене Unity или в WPF-приложении.

## 🔄 Дальнейшее развитие

Проект будет последовательно перенесён на различные интерфейсы **без изменений в ядре**:

- 🖥️ Консоль (текущая реализация — для тестов)
- 🪟 WinForms
- 🖌️ WPF
- 🎮 Unity

Это наглядно демонстрирует преимущества грамотного разделения архитектуры.


| Итерация | Проект | Архитектура |
|----------|--------|-------------|
| **1** | **Console App** *(текущая версия)* | Ядро игры, разделение логики и интерфейса |
| 2 | [WinForm V_0](https://github.com/VladimirRepp/WinForms---Samples/tree/main/SimpleSeaBatlle/SimpleSeaBattle%20-%20WinForm%20V0) | Быстрый перенос без паттернов |
| 3 | [WinForms MVP](https://github.com/VladimirRepp/WinForms---Samples/tree/main/SimpleSeaBatlle/SimpleSeaBattle%20-%20WinForm%20MVP) | Model-View-Presenter |
| 4 | [Unity](https://github.com/VladimirRepp/Unity---SimpleSeaBattle) | Пример переноса логики на Unity |
